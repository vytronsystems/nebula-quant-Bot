# NEBULA-QUANT v1 | Weekly Audit Standard

## Objetivo
Revisar todas las operaciones y todos los experimentos de la semana para mejorar el sistema.

## Frecuencia
Semanal

## Revisar siempre por cada trade
- entradas
- salidas
- TP
- SL
- RR esperado
- RR real
- slippage
- setups ganadores
- setups perdedores
- condiciones de mercado
- errores operativos
- edge decay
- duracion del trade

## Resultado esperado
El weekly audit debe producir:
- hallazgos
- cambios sugeridos
- estrategias a ajustar
- estrategias a pausar
- nuevas hipótesis a investigar

## Reglas del sistema

No se mejora el sistema por intuición.

Toda mejora debe:

1. quedar documentada
2. convertirse en hipótesis
3. volver a validarse mediante backtesting

## Análisis de estrategias

Evaluar:

- setups ganadores
- setups perdedores
- consistencia del edge
- cambios en comportamiento del mercado
- edge decay


## Métricas clave

Calcular siempre:

- win rate
- profit factor
- average RR
- max drawdown
- expectancy
- average slippage

## Revisión de ejecución

Detectar:

- errores operativos
- problemas de ejecución
- latencia
- desviaciones de precio

---

## Revisión de experimentos

Evaluar todos los experimentos activos:

- hipótesis probadas
- hipótesis rechazadas
- resultados inconsistentes
- necesidad de nuevos tests

---

## Identificación de problemas

Detectar:

- estrategias deterioradas
- cambios estructurales de mercado
- setups que dejaron de funcionar
- errores del sistema

## Flujo de mejora

weekly audit  
→ nueva hipótesis  
→ backtest  
→ walkforward  
→ paper  
→ promoción si es válida


## Principio fundamental

El sistema mejora **mediante evidencia, no opiniones.**
